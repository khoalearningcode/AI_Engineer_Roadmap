Project Ladder

Six projects that prove the roadmap

Each project should produce a GitHub README with architecture, benchmark evidence, and a clear NVIDIA relevance note.

Project 1

Enterprise RAG System

Goal: Build a production-style RAG app for enterprise documents.

Tech stack: FastAPI, vector DB, reranker, eval set, logs.

Definition of done: Citations, recall@k, latency report, failure cases.

README: Architecture diagram, eval table, sample queries.

NVIDIA relevance: Enterprise GenAI and Retriever-style patterns.

Project 2

vLLM Inference Benchmark

Goal: Benchmark self-hosted LLM serving.

Tech stack: vLLM, OpenAI-compatible API, benchmark script.

Definition of done: TTFT, TPOT, tokens/sec, memory report.

README: Hardware config, commands, benchmark chart.

NVIDIA relevance: GPU-aware inference and cost per token thinking.

Project 3

Mini NIM-like AI Service

Goal: Package an AI model behind a clean service API.

Tech stack: FastAPI, Docker, model server, metrics endpoint.

Definition of done: Health checks, streaming, config, logs.

README: API contract, container runbook, service diagram.

NVIDIA relevance: NIM-style deployable inference service concept.

Project 4

Multimodal Agentic RAG

Goal: Add image/document understanding to agentic retrieval.

Tech stack: CLIP/SigLIP, OCR, VLM prompting, vector search.

Definition of done: Top-k visual retrieval, grounded answers, examples.

README: Visual demos, retrieval metrics, limitations.

NVIDIA relevance: Multimodal enterprise AI and vision AI agents.

Project 5

Triton/TensorRT Deployment Demo

Goal: Deploy one model through an optimized inference path.

Tech stack: ONNX, Triton, TensorRT basics, batching notes.

Definition of done: Before/after latency and memory comparison.

README: Model config, commands, latency table.

NVIDIA relevance: Direct inference platform alignment.

Project 6

Vision AI / Physical AI Preview

Goal: Show awareness of simulation, synthetic data, and 3D perception direction.

Tech stack: image/video dataset, 3D perception notes, synthetic data summary.

Definition of done: Small demo plus system design write-up.

README: Physical AI concept map and future extension plan.

NVIDIA relevance: Cosmos/Isaac Sim awareness without overclaiming robotics depth.

Positioning Pitch

AI Systems Engineer focused on GenAI platforms, GPU inference, and multimodal AI systems.

Copy-ready pitch

I build production-style LLM/RAG systems with retrieval, reranking, evaluation, streaming generation, and observability. I am expanding toward GPU-aware inference using vLLM, quantization, Triton/TensorRT concepts, and NVIDIA-style AI platform design. My differentiator is the ability to connect GenAI systems with multimodal and vision AI workflows, preparing for enterprise AI and physical AI applications.

Portfolio proof points

  • Enterprise RAG system with retrieval metrics, citations, logs, and failure analysis.
  • vLLM inference benchmark with latency, throughput, GPU memory, and cost per token framing.
  • Mini NIM-like AI service with API, containerization, health checks, and observability.
  • Multimodal RAG or physical AI preview connecting vision workflows to future AI systems.

Built as an NVIDIA-aligned roadmap: enterprise GenAI → GPU inference → AI platform → multimodal intelligence → physical AI readiness.
