Project Ladder

Build portfolio proof in order

Each project should leave behind a GitHub README that explains architecture, trade-offs, benchmarks, and screenshots.

Project 1

Production RAG System

Goal

Turn a chatbot demo into a measurable document QA system.

Tech stack

FastAPI, vector DB, reranker, structured logs, eval set.

Definition of done

Citations, retrieval logs, recall@k, latency, failure cases.

Show in README

Architecture diagram, eval table, sample questions, screenshots.

Project 2

vLLM Inference Benchmark

Goal

Benchmark local model serving instead of only calling hosted APIs.

Tech stack

vLLM, OpenAI-compatible API, GPU metrics, benchmark script.

Definition of done

TTFT, TPOT, tokens/sec, memory report, streaming demo.

Show in README

Benchmark chart, hardware config, commands, trade-offs.

Project 3

Mini GenAI Platform

Goal

Combine RAG, model serving, API gateway, and observability.

Tech stack

FastAPI, Docker Compose, PostgreSQL, Redis, vector DB, dashboard.

Definition of done

Services run locally, logs are inspectable, config is documented.

Show in README

Service diagram, API docs, runbook, dashboard screenshots.

Project 4

Multimodal RAG Extension

Goal

Extend GenAI beyond text with image or document understanding.

Tech stack

CLIP/SigLIP, OCR, VLM prompting, top-k retrieval, eval metrics.

Definition of done

Image-text retrieval works with visible examples and recall@k.

Show in README

Visual demo, retrieval examples, metric table, limitations.

Positioning Pitch

AI Engineer focused on production RAG, LLM serving, and GenAI platform systems.

Short pitch

I build production-style LLM/RAG systems with document ingestion, vector retrieval, reranking, streaming generation, evaluation, and monitoring. I am expanding toward GPU-aware inference using vLLM and platform-level GenAI deployment with clear reliability practices.

Proof points

  • Production RAG system with retrieval quality metrics, citations, and failure analysis.
  • vLLM serving benchmark with latency, throughput, and GPU memory reporting.
  • Mini GenAI platform with gateway API, model server, vector database, logs, and dashboard.
  • Optional multimodal extension using CLIP/SigLIP-style image-text retrieval or document AI.

Built as a portfolio-ready AI Engineer career roadmap: production RAG → LLM serving → GenAI platform → specialization branches.
